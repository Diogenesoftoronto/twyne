# Downloads

> Take Twyne to your own desk — or keep it in the browser.

The desktop app is a thin native shell around the live workspace at
twyne.love — sync, hosted AI, publishing, and sign-in all behave the same as
the web. Pick a platform below, or simply keep writing in your browser.
Either way, your local-first folios stay yours.

## Desktop app

Native builds for macOS, Windows, and Linux. Each download is published to
the latest GitHub release and opens Twyne in its own window with deep links
for ATProto sign-in.

- **macOS** — Apple silicon & Intel, `.tar.gz`. Universal native shell. Unzip
  and drag Twyne to Applications. [Download for macOS](/download/macos)
- **Windows** — Windows 10/11, `.tar.gz`. Native window over the live app.
  Extract and run the bundled executable.
  [Download for Windows](/download/windows)
- **Linux** — x86_64, `.tar.gz`. Native shell for most modern distributions.
  Extract and launch. [Download for Linux](/download/linux)

Builds are unsigned for now, so your operating system may ask you to confirm
before launching an app from an unidentified developer. The desktop shell
never bundles your writing — it loads the same hosted workspace you use on
the web.

## Web & install

No download required. Twyne runs in any modern browser and can be installed
as a Progressive Web App for a standalone window, dock icon, and
offline-friendly local-first storage.

- **Open in the browser** — Visit Twyne and start writing immediately — no
  account needed. To install, use your browser's "Install app" or "Add to
  Home Screen" option from the address bar or menu. [Launch Twyne](/)

## Your data travels with you

- Local-first by default — your brief, folios, and drafts live in your
  browser's IndexedDB until you choose to sync.
- Sign in on any build to sync the same projects across desktop and web.
- Bring your own AI key, or use hosted AI — both work identically across
  platforms.
- Export any folio as Markdown, HTML, text, or a `.twyne.json` backup, on
  any device.
